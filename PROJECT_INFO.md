# Goals Widget & Future Canvas — Системная спецификация проекта

Данный документ представляет собой полное описание архитектуры, используемого технологического стека, структуры модулей и логики взаимодействия компонентов проекта. Он предназначен для контекстуализации внешних AI-агентов или разработчиков.

---

## 1. Общая концепция проекта
Проект представляет собой **многофункциональный десктопный виджет и рабочую станцию** для Windows. Он объединяет финансовое планирование, отслеживание личных целей, ведение бэклога задач по методологии Obsidian (в `.md` файлах), торговый журнал для трейдеров и экспериментальный холст временного планирования (Future Canvas).

Приложение разработано для плотной интеграции с локальной базой знаний (Obsidian Vault) и внешними финансовыми сервисами.

---

## 2. Архитектура и структура модулей

Приложение спроектировано как **Electron-приложение** с мультиоконной архитектурой (один главный процесс управления и несколько окон-рендереров).

### Основные окна и их назначение:
1. **Главный виджет (Main Window)** (`index.html`, `script.js`, `styles.css`):
   * Визуализация финансовой цели, прогресса ее достижения и расчета необходимых регулярных взносов с учетом доходности капитала.
   * Отображение и интерактивное управление бэклогом задач и списком ежедневных дел.
2. **Таймер перерывов (Break Window)** (`break.html`):
   * Полноэкранный блокирующий интерфейс, напоминающий о необходимости делать регулярные перерывы для отдыха.
   * Поддерживает отображение медиаконтента (картинки, видео, звук) и может гибко отключаться/настраиваться в параметрах приложения.
3. **Модуль аналитики (Analytics Window)** (`analytics.html`):
   * Дашборд с графиками продуктивности и историей выполнения задач по дням.
4. **Дневник трейдинга (Trading Diary)** (`app/frontend.html`, `app/trading-journal.jsx`):
   * Полноценный журнал сделок для фиксации торговых сессий.
   * Автоматический расчет параметров риска (риск на сделку, лимит на месяц).
   * Интеграция с локальным выбором скриншотов графиков сделок.
5. **Мои связи / Картотека** (`app/connections.html`, `app/connections.js`):
   * Менеджер контактов и отношений на базе Markdown-карточек Obsidian.
   * Автоматическое создание карточек с инкрементным ID (`PER-XXX`).
   * Поддержка кликабельных Wiki-ссылок типа `[[Имя Контакта]]` для быстрой навигации.
6. **Future Canvas (Карта будущего)** (`app/future-canvas/future-canvas.html`, `app/future-canvas/future-canvas.js`):
   * Интерактивный бесконечный холст планирования вероятных сценариев будущего.
   * Зуммируемая ось времени (от дней до десятилетий) в стиле графиков TradingView.
   * Узлы трех типов (Событие, Решение, Цель), соединенные кривыми линиями (толщина зависит от вероятности).
7. **AlgoTrading (Панель автоторговли)** (`app/AlgoTrading/frontend.html`, `app/AlgoTrading/backend.py`):
   * Управление и мониторинг объединенной платформы отслеживания Heikin Ashi ценовых сигналов (ранее PriceAlert) и автоматического сеточного торгового робота (ранее Seller).
   * Работает полностью автономно в фоновом режиме, запускаясь при старте виджета.
   * Безрамочное стилизованное окно с кастомным drag-заголовком (полный визуальный аналог Future Canvas).

## 3. Технологический стек (Tech Stack)

### Ядро и системный уровень:
* **Electron (`^28.0.0`)**: Хост приложения, управление жизненным циклом окон, интеграция с ОС (горячие клавиши, системные диалоги).
* **Node.js LTS**: Выполнение файловых операций, запуск дочерних процессов (Git CLI).

### Фронтенд (Интерфейсы):
* **HTML5 / CSS3 (Vanilla)**: Основная верстка окон. Стилизация выполнена в темных тонах (Technical Minimalism/Cyberpunk-смесь Obsidian Graph View и биржевых терминалов). Используется много полупрозрачных элементов, тонких рамок и размытия фона (`backdrop-filter`).
* **JavaScript (Vanilla ES6)**: Интерактивная логика окон, вычисления формул сложных процентов, обработка событий холста Future Canvas.
* **React (JSX)**: Используется локально в Дневнике Трейдинга ([trading-journal.jsx](file:///D:/Documents/Programming/Goals_Wiget/app/trading-journal.jsx)) для реактивного отображения сложных табличных данных и форм ввода сделок.
* **Шрифты**: `JetBrains Mono` (для цифр, кодов, меток терминала) и `Inter` (для основного текста).

## 4. Интеграции и потоки данных

### 1. Межпроцессное взаимодействие (Electron IPC)
Фронтенд общается с системным процессом через безопасный мост `preload.js`.
* **Renderer $\rightarrow$ Main**: Отправка событий (запись файла, открытие нового окна, логирование действий в `log.txt`, запуск Git-команд).
* **Main $\rightarrow$ Renderer**: Передача обновленных данных из файлов, конфигураций или ответов API.

### 2. Obsidian Integration (Файловая база данных)
* Приложение не использует традиционные СУБД (PostgreSQL, SQLite). Базой данных служат обычные текстовые `.md` файлы в локальной папке Obsidian Vault.
* Путь к папке настраивается в [main.js](file:///D:/Documents/Programming/Goals_Wiget/main.js) константой `TARGET_DIR`.
* Формат задач: строго `- [ ] Задача` и `- [x] Выполненная задача`. Вложенность подзадач определяется отступами.

### 3. Автоматическая Git-синхронизация
* Процесс [main.js](file:///D:/Documents/Programming/Goals_Wiget/main.js) автоматически запускает асинхронный `git pull` при старте приложения для синхронизации папки Obsidian.
* При внесении изменений в задачи выполняется автокоммит и автопуш с механизмом **дебаунса (5 секунд)** для объединения частых мелких изменений и снижения нагрузки на Git-сервер.

### 4. Финансовые API
* Интеграция с **Т-Банк (T-Investments API)** для получения текущих биржевых котировок акций и фьючерсов. Токен авторизации хранится локально в файле `config.json` и передается в заголовках IPC-запросов.

## 5. Важные архитектурные ограничения (Constraints)

1. **Неблокирующий UI**: Все тяжелые вызовы (внешние запросы, выполнение команд Git через CLI) должны выполняться асинхронно, чтобы не вызывать "зависание" интерфейса виджета.
2. **Сохранение структуры Markdown**: При перезаписи `.md` файлов парсер обязан строго сохранять исходное форматирование пользователя, включая переводы строк, пустые строки и оригинальные табы/пробелы.
3. **Безопасность учетных данных**: Персональные API-токены и токены авторизации Git в URL репозитория вырезаются при записи в публичный файл журнала `log.txt`.
4. **Совместимость путей (Windows)**: Все системные вызовы командной строки учитывают наличие кириллических символов и пробелов в путях к файлам (пути всегда экранируются/оборачиваются в кавычки).

## 6. Как запустить проект в режиме разработки

1. Убедиться в наличии установленных **Node.js** и **Git**.
2. В терминале выполнить:
   ```bash
   npm install
   ```
3. Для запуска Electron-приложения:
   ```bash
   npm start
   ```
4. Для сборки дистрибутива (инсталлятора `.exe` через `electron-builder`):
   ```bash
   npm run build
   ```
