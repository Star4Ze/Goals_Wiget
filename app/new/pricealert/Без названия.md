## 🔴 БЛОК 1 — Починить графики (критично)

### 1.1 — Синхронизация текущей цены на графике

**Проблема:** График прогружает начальные данные но не обновляется в реальном времени.

**Что сделать:**
- В `frontend.html` WebSocket-обработчик при получении новой цены должен:
  1. Найти последнюю свечу на графике
  2. Обновить её `close`, `high`, `low` в зависимости от новой цены
  3. Вызвать `chart.update('none')` (без анимации — для производительности)
- Если новая свеча (новый период) — добавить  точку в dataset и удалить первую (sliding window)
- Для HA-режима — пересчитывать последнюю HA-свечу при каждом обновлении цены

### 1.2 — Подгрузка истории при прокрутке влево (lazy loading)

**Проблема:** При прокрутке влево новые свечи не догружаются.

**Что сделать:**

**Backend (`api.py`):**
- Добавить эндпоинт `GET /candles/{symbol}?tf={timeframe}&before={timestamp}&limit=200`
- Логика: сначала смотреть в `candles.db` — если есть данные до этого timestamp, отдать из БД. Если нет — запросить у Tinkoff/MOEX, сохранить в БД, отдать клиенту

**Frontend (`frontend.html`):**
- Повесить обработчик на событие `pan` или `scroll` графика Chart.js
- Когда пользователь доскроллил до левого края (первая видимая свеча) — отправить запрос на бэкенд с `before = timestamp первой свечи`
- Полученные свечи prepend в начало dataset
- Флаг `isLoadingHistory = true` во время загрузки (чтобы не дублировать запросы)

### 1.3 — Умное кэширование свечей в БД

**Проблема:** БД не используется как полноценный кэш — каждый раз идёт запрос к API.

**Что сделать в `db.py` и `tinkoff.py` / `moex.py`:**
- При запросе свечей сначала проверять `candles.db` — есть ли данные для `symbol + timeframe + диапазон дат`
- Если в БД есть данные — отдавать из БД
- Если нет или данные устарели (последняя свеча старше 1 периода) — догружать только недостающий диапазон из API
- При сохранении — использовать `INSERT OR IGNORE` по `(symbol, timeframe, timestamp)` как уникальный ключ
- Структура таблицы `candles`:
```sql
CREATE TABLE IF NOT EXISTS candles (
    symbol      TEXT NOT NULL,
    timeframe   TEXT NOT NULL,
    ts          INTEGER NOT NULL,
    open        REAL,
    high        REAL,
    low         REAL,
    close       REAL,
    volume      REAL,
    PRIMARY KEY (symbol, timeframe, ts)
);
```

### 1.4 — Предзагрузка истории при добавлении инструмента в watchlist

**Что сделать:**
- При добавлении актива в watchlist — фоновая задача загружает сразу все таймфреймы (1m, 5m, 15m, 1H, 4H, 1D) за максимально доступный период
- Загрузка батчами с задержкой между запросами (не нарушать rate limit Tinkoff: не более 300 запросов/мин)
- Прогресс отображать в UI (например, "Загрузка истории: 1H... 67%")

**Проблема:** Алерты не срабатывают.

**Что проверить и починить в `alerts.py`:**

- Проверить что `previous_price` корректно хранится между тиками (не None)

### 2.2 — Цена на линии алерта — при наведении, справа

### 3.2 — Новые типы триггеров в `ha_analyzer.py`

Добавить проверку трёх новых условий помимо MTF-сигнала:

**Триггер 1 — Разворот по HA:**
- Уже частично есть в `ha_analyzer.py` через `REVERSAL_BULL` / `REVERSAL_BEAR`
- Добавить настройку минимального количества свечей для подтверждения (параметр `min_candles`, по умолчанию 3)
- Алерт отправляется только при смене сигнала (не повторять каждые 5 минут)

**Триггер 2 — Прострел цены (Price Spike):**
```python
def check_price_spike(candles, period_tf, threshold_pct):
    # Берём последнюю закрытую свечу
    last = candles[-2]  # -1 это текущая незакрытая
    prev = candles[-3]
    change_pct = abs(last['close'] - prev['close']) / prev['close'] * 100
    if change_pct >= threshold_pct:
        direction = 'UP' if last['close'] > prev['close'] else 'DOWN'
        return {'triggered': True, 'change_pct': change_pct, 'direction': direction}
    return {'triggered': False}
```

**Триггер 3 — Аномальный объём (Volume Spike):**
```python
def check_volume_spike(candles, multiplier):
    vol_sma20 = sum(c['volume'] for c in candles[-22:-2]) / 20
    last_vol = candles[-2]['volume']
    vol_ratio = last_vol / vol_sma20 if vol_sma20 > 0 else 0
    return {'triggered': vol_ratio >= multiplier, 'vol_ratio': round(vol_ratio, 2)}
```

**Триггер 4 — Комбо (HA разворот + объём):**
- Срабатывает только если одновременно `REVERSAL_BULL/BEAR` И `vol_ratio >= threshold`
- Самый сильный сигнал, отдельный тип уведомления
