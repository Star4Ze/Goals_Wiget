# Архитектура и техническое описание приложения PriceAlert (Heikin Ashi & Alerts)

Данный документ представляет собой полное техническое описание архитектуры, назначения файлов, логики работы и порядка запуска приложения **PriceAlert** в экосистеме **Lumi**.

---

## 📌 Общая концепция приложения

**PriceAlert** — это интерактивный торговый терминал и система мониторинга котировок в реальном времени. Приложение решает три ключевые задачи:
1. **Визуализация котировок**: построение интерактивных ценовых графиков (свечи/линии) с поддержкой технических индикаторов (Chart.js).
2. **Система алертов**: отслеживание пересечения цен с горизонтальными уровнями и наклонными трендовыми линиями, заданными пользователем, с отправкой системных уведомлений.
3. **HA Trend Analyzer**: фоновый автоматический многотаймфреймовый (Multi-Timeframe — MTF) анализ трендов по системе Heikin Ashi на таймфреймах 1H и 4H с мгновенной WebSocket-трансляцией сигналов и алертов при изменении тренда.

## 📂 Структура файлов и их роли

```
apps/pricealert/
├── backend.py            # Точка входа для запуска FastAPI бэкенда
├── manifest.json         # Метаданные mini-app (версия, порты, права)
├── config.json           # Файл конфигурации (сервер, базы данных, токен Tinkoff)
├── requirements.txt      # Зависимости Python
├── frontend.html         # Single Page Application (UI: HTML5, CSS3, JS, Chart.js)
├── icon.png              # Иконка приложения для дашборда Lumi
└── logic/
    ├── __init__.py       # Регистрация модуля
    ├── api.py            # Маршрутизация FastAPI, WebSocket и логика соединений
    ├── db.py             # Создание схем SQLite (user, registry, candles) и CRUD
    ├── alerts.py         # Ядро проверки алертов и отправка системных уведомлений
    ├── ha_analyzer.py    # Фоновый анализатор Heikin Ashi (расчеты, тренды, MTF)
    ├── tinkoff.py        # Клиент Tinkoff Invest API (загрузка свечей, последние цены)
    └── moex.py           # Клиент MOEX ISS API (загрузка свечей по российским акциям)
```

### Подробное описание каждого файла:

#### 1. `backend.py`
Точка входа. Считывает файл `config.json`, настраивает логгирование в локальный файл `data/pricealert.log` и запускает ASGI-сервер `uvicorn` на порту, указанном в переменной окружения `PORT` (или берет порт по умолчанию из конфига — `8765`).

#### 2. `manifest.json`
Файл-манифест, считываемый менеджером приложений `Lumi Core` (`main.py` в корне проекта). Содержит имя, ID, порт для запуска бэкенда (`8765`), флаг автостарта `launch_on_start: true` и права доступа.

#### 3. `logic/db.py`
Управляет тремя независимыми базами данных SQLite:
* **`candles.db`** — кэш исторических свечей по инструментам во избежание частых запросов к внешним API (Tinkoff / MOEX).
* **`registry.db`** — реестр доступных торговых инструментов (акции, валюта, фонды), синхронизируемый при старте.
* **`pricealert.db`** (пользовательская БД, по умолчанию `Admin`) — хранит:
  - `alerts` — установленные горизонтальные и наклонные алерты (активность, цены, время, направление пересечения).
  - `alert_log` — историю сработавших уведомлений.
  - `lines` — сохраненные графические линии разметки графика.
  - `watchlist` — избранные инструменты пользователя.
  - `chart_settings` — настройки индикаторов и масштаба графиков.

#### 4. `logic/tinkoff.py`
Реализует асинхронный REST-клиент для Tinkoff Invest API. Использует `httpx` для отправки POST-запросов к эндпоинтам Tinkoff (загрузка исторических свечей `GetCandles` и получение текущих цен `GetLastPrices` пачками). Выполняет парсинг дробных цен (поля `units` и `nano`).

#### 5. `logic/moex.py`
Клиент для взаимодействия с бесплатным API Московской биржи (MOEX ISS). Автоматически определяет, торгуется ли инструмент на MOEX, загружает исторические свечи через REST API биржи, форматирует их в стандартную структуру и кэширует.

#### 6. `logic/alerts.py`
* **Движок проверки условий (Alert Engine)**: содержит функцию `check_alert(alert, current_price, previous_price)`. Для наклонных трендовых линий вычисляет проекцию цены на текущую секунду по формуле линейной интерполяции:
  $$P_{target} = P_1 + \frac{(P_2 - P_1) \cdot (T_{now} - T_1)}{T_2 - T_1}$$
* **WebSocket Manager**: координирует трансляцию текущих цен и сигналов клиентам по адресам `/ws/{nick}`.
* **Интеграция уведомлений**: функция `send_native_notification` генерирует JWT-токен авторизации (подписанный системным `JWT_SECRET_KEY`) и отправляет HTTP POST запрос на адрес центрального ядра `Lumi Core` (`/api/apps/notifications/send`), который доставляет пуш-уведомление пользователю в браузер, Telegram или шторку телефона.

#### 7. `logic/ha_analyzer.py`
Фоновый демон Heikin Ashi Trend Analyzer.
* Работает в бесконечном цикле планировщика (по умолчанию раз в 300 секунд / 5 минут).
* Для каждого отслеживаемого инструмента загружает 100 свечей для таймфреймов `1H` (1 час) и `4H` (4 часа).
* Рассчитывает свечи Heikin Ashi по классическим формулам:
  - $HA\_Close = \frac{Open + High + Low + Close}{4}$
  - $HA\_Open = \frac{HA\_Open_{prev} + HA\_Close_{prev}}{2}$
  - $HA\_High = \max(High, HA\_Open, HA\_Close)$
  - $HA\_Low = \min(Low, HA\_Open, HA\_Close)$
* **Trend Engine**: анализирует серию свечей одного цвета, оценивает наличие и размеры теней (хвостов), динамику тел свечей и сопоставляет объем свечи со скользящей средней SMA(20) по формуле `vol_ratio = vol / vol_sma20`.
* **MTF Матрица**: комбинирует тренды 1H и 4H в один из сигналов:
  - `CONFIRMED_UP` (оба TF растут с сильным объемом)
  - `CONFIRMED_DOWN` (оба TF падают с объемом)
  - `PULLBACK_END_UP` (завершение коррекции вверх)
  - `PULLBACK_END_DOWN` (завершение коррекции вниз)
  - `CONFLICT` (разнонаправленные тренды)
  - `NO_SIGNAL` (флэт / неопределенность)
* При изменении MTF-сигнала генерирует событие в WebSocket-канал и вызывает `send_native_notification` для отправки пуша.

#### 8. `logic/api.py`
Настраивает приложение `FastAPI`. Инициализирует базы данных, запускает фоновые задачи (`stream_loop` для стриминга цен и `ha_analyzer.start()` для Heikin Ashi). Содержит HTTP-руты для управления алертами, линиями, watchlist и конфигурации HA.

#### 9. `frontend.html`
Сверхмощный SPA-интерфейс на ванильном HTML/JS/CSS.
* Отображает интерактивный график Chart.js.
* Позволяет переключаться между стандартными японскими свечами и Heikin Ashi (все расчеты HA-режима графика происходят на лету на стороне клиента в JS).
* Поддерживает визуальное рисование горизонтальных (клик по графику) и наклонных (перетаскивание) линий алертов.
* Реализует WebSocket-клиент для мгновенного обновления цены инструмента, анимирования графика в реальном времени и отображения тостов при срабатывании алертов.
* Включает вкладку **📊 HA**, выводящую карточки сигналов Heikin Ashi, обратный отсчет до следующего автоматического анализа, лог событий и панель управления отслеживаемыми инструментами.

## 🚀 Как запускается приложение

### 1. Требования к окружению (Python)
Для работы бэкенда необходимы пакеты, перечисленные в `requirements.txt`:
- `fastapi`, `uvicorn`, `httpx`, `pydantic`, `websockets`, `pyjwt`.

Установка:
```bash
pip install -r requirements.txt
```

### 2. Конфигурация (`config.json`)
Перед запуском необходимо прописать действующий токен Tinkoff Invest API в поле `token`:
```json
{
  "tinkoff": {
    "token": "t.your_tinkoff_api_token_here",
    "base_url": "https://invest-public-api.tinkoff.ru/rest"
  },
  "server": {
    "port": 8765
  },
  "data_dir": "data/{nick}"
}
```

### 3. Запуск вручную
Перейдите в каталог приложения и выполните скрипт:
```bash
cd apps/pricealert
python backend.py
```
После этого:
- Бэкенд запустится на порту `8765` (или из `PORT`).
- API будет доступно по адресу `http://localhost:8765/`.
- Интерфейс автоматически загрузится по корневому адресу `/`.

### 4. Запуск в составе Lumi (Docker Compose)
При интеграции в общую среду Docker Compose проекта Lumi, запускается управляющий модуль `app-manager` (Lumi Core). Он считывает `manifest.json`, автоматически создает изолированный докер-контейнер для бэкенда `pricealert` с пробросом нужных портов и автоматически запускает его при старте системы, направляя трафик с дашбора на порт бэкенда.
Пользователь заходит в дашборд Lumi, нажимает на иконку «Price Alert» и дашборд открывает SPA-страницу `frontend.html`, передавая в URL параметры авторизации, lang и API хост.
